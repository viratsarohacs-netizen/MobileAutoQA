# Test Generation Agent

**Role:** Converts extracted requirements into plain-English test cases, then into
runnable pytest classes.

This is **Agent 2 of 5**:
> Requirements Agent → **Test Generation Agent** → Execution Agent → Self-Healing Agent → Reporting Agent

---

## Usage

```
/test-generation-agent PHIX-97533
/test-generation-agent PHIX-97533 --platform=both
```

Input: `tests/jira/{JIRA-ID}/requirements.md` (from Requirements Agent).
If it doesn't exist, run `/requirements-agent {JIRA-ID}` first.

---

## Step 1 — Plan Coverage

Read `requirements.md`. Build a coverage table BEFORE writing any test:

| TC | Scenario | Type | Priority | Screen | Maps to AC/FR |
|----|----------|------|----------|--------|---------------|
| TC-01 | Happy path: {flow} | Normal | High | Time Tracking | AC-1 |
| TC-02 | Validation: {error} | Edge | Med | Time Tracking | AC-3 |
| TC-R01 | Regression: clock in still works | Regression | High | Time Tracking | — |

**Coverage rules:**
- Every Acceptance Criterion → at least one test case
- Every new button/label → a tap/verify test
- Every error message → exact-text assertion
- Every popup/modal → "appears" and "dismissed" paths
- Every conditional/edge behavior → its own test
- Core flows the ticket touches → add Regression cases
- Minimum 10 cases for a defect, 20+ for a story

---

## Step 2 — Write Plain-English Spec

Write `tests/jira/{JIRA-ID}/mobile.test.md` using the controlled vocabulary
(same as `tests/sanity/sanity.test.md`):

```markdown
# {JIRA-ID} — {Summary}
# Platform: both | android | ios
# Screens: {list}

## Normal Cases

### TC-01: {name}
**What this tests:** {one sentence}
**Precondition:** {state}
**Platform:** both

#### SETUP
tap "Time Tracking"
[steps to reach precondition]

#### TEST
tap "Clock In"
dismiss popup if present: "Close"
verify clocked in
screenshot "tc-01"

#### TEARDOWN
tap "Clock Out"
dismiss attestation if present
```

**Step vocabulary** (executor + generator both understand these):

| Step | Page-object call |
|------|------------------|
| `tap "X"` | `page.tap("X")` |
| `enter "V" into field[N]` | `page.enter_by_index(N, "V")` |
| `enter "V" into "Field"` | `page.enter("Field", "V")` |
| `verify "X" is visible` | `page.verify_visible("X")` |
| `verify "X" is not visible` | `page.verify_not_visible("X")` |
| `verify clocked in` | `assert tt.is_clocked_in()` |
| `verify clocked out` | `assert tt.is_clock_in_button_visible()` |
| `wait for "X" to appear` | `page.wait_for_text("X")` |
| `dismiss popup if present: "X"` | `page.dismiss_popup_if_present("X")` |
| `dismiss attestation if present` | `dashboard.dismiss_attestation_if_present()` |
| `open drawer` | `dashboard.open_drawer()` |
| `scroll to "X"` | `page.scroll_to_text("X")` |
| `press back` | `page.press_back()` |
| `screenshot "label"` | (reporter auto-captures; explicit shots optional) |

**Use real app text** from `core/locators.py` — the app has no testIDs.

---

## Step 3 — Generate pytest Class

Write `tests/jira/{JIRA-ID}/test_{jira_id_lower}.py`:

```python
"""
Auto-generated by Test Generation Agent.
Source: tests/jira/PHIX-97533/mobile.test.md
Regenerate with: /test-generation-agent PHIX-97533
"""
import pytest
from pages.dashboard_page import DashboardPage
from pages.time_tracking_page import TimeTrackingPage
from core.locators import TimeTracking


@pytest.mark.jira
@pytest.mark.usefixtures("driver")
class TestPHIX97533:

    def _begin(self, test_id, name):
        self._current_test_id = test_id
        self._current_test_name = name
        self._heal_log = []

    def _collect(self, *pages):
        for p in pages:
            self._heal_log.extend(getattr(p, "heal_log", []))

    def test_tc01_{snake_name}(self):
        self._begin("TC-01", "{name}")
        dash = DashboardPage(self.driver)
        tt = dash.open_time_tracking()
        # SETUP
        tt.ensure_clocked_out()
        # TEST
        tt.clock_in()
        tt.wait_for_clocked_in(90)
        assert tt.is_clocked_in()
        # TEARDOWN
        tt.clock_out()
        self._collect(dash, tt)
```

**Generation rules:**
- Each `### TC-NN` block → one `test_tcNN_*` method
- `self._begin("TC-NN", "name")` first line of every test (drives the reporter)
- SETUP steps before assertions; TEARDOWN in normal flow or `finally`
- `self._collect(...)` last line so heal logs reach the report
- Reuse existing page objects; if a step needs a new interaction, ADD a method to
  the relevant page object (e.g. `TimeOffPage.submit_request`) rather than inlining raw Appium
- If a NEW locator is needed, add it to `core/locators.py` and reference it

---

## Step 4 — If New Page/Locator Coverage Is Needed

When the ticket touches a screen with no page object yet:
1. Create `pages/{screen}_page.py` extending `BasePage`
2. Add the screen's text constants to `core/locators.py`
3. Keep methods small and intention-revealing
